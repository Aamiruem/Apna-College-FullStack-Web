JavaScript DOM Events Tutorial

Learn how to handle events in the Document Object Model

What are DOM Events?

DOM events are actions that occur as a result of user interaction or other activities in the browser. JavaScript can detect these events and execute code in response.

Events allow you to create dynamic, interactive web applications by responding to user actions like clicks, keyboard input, mouse movements, and more.

Common Event Types

  • Mouse Events: click, dblclick, mousedown, mouseup, mousemove, mouseover, mouseout
  • Keyboard Events: keydown, keyup, keypress
  • Form Events: submit, change, focus, blur, input
  • Window Events: load, resize, scroll, beforeunload
  • Touch Events: touchstart, touchmove, touchend

Event Handling Methods

Inline Event Handlers

<button onclick="handleClick()">
  Click Me
</button>

<script>
  function handleClick() {
    alert('Button clicked!');
  }
</script>

Simple but not recommended for complex applications (mixes HTML and JavaScript).

Event Listeners

<button id="myBtn">Click Me</button>

<script>
  const button = document.getElementById('myBtn');
  button.addEventListener('click', function() {
    alert('Button clicked!');
  });
</script>

Recommended approach - separates structure from behavior and allows multiple handlers.

Event Object

When an event occurs, the browser creates an event object containing information about the event. This object is passed to the event handler.

element.addEventListener('click', function(event) {
  console.log(event.type); // Event type (e.g., 'click')
  console.log(event.target); // Element that triggered the event
  console.log(event.clientX, event.clientY); // Mouse coordinates
  event.preventDefault(); // Prevent default behavior
  event.stopPropagation(); // Stop event bubbling
});

The event object provides valuable information about the event and methods to control its behavior.

Event Propagation: Bubbling and Capturing

Event Bubbling

Events bubble up from the target element to the root:

document.getElementById('child').addEventListener('click', function() {
  console.log('Child clicked');
});

document.getElementById('parent').addEventListener('click', function() {
  console.log('Parent clicked');
});

When the child is clicked, both handlers will fire (child first, then parent).

Event Capturing

Events can be captured from root to target (less common):

document.getElementById('parent').addEventListener('click', function() {
  console.log('Parent captured');
}, true); // Use capture phase

document.getElementById('child').addEventListener('click', function() {
  console.log('Child clicked');
});

With capturing, the parent handler fires before the child handler.
