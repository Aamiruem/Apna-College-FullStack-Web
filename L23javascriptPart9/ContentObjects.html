JavaScript innerText vs innerHTML vs textContent

Try It Yourself

Enter some text with HTML tags (like <b>, <i>, <u>) and see how each property handles it:

innerText

element.innerText = "Hi, I am <b>Peter Parker</b>!";

Renders text as it appears visually, ignoring HTML tags.

innerHTML

element.innerHTML = "Hi, I am <b>Peter Parker</b>!";

Parses and renders HTML content, interpreting tags.

textContent

element.textContent = "Hi, I am <b>Peter Parker</b>!";

Gets or sets the raw text content, ignoring formatting.

Key Differences

  • innerText is aware of styling and doesn't render hidden elements. It returns how text would be rendered visually.
  • innerHTML parses and renders HTML tags, allowing you to insert actual HTML elements.
  • textContent gets or sets the raw text content without any formatting awareness.
